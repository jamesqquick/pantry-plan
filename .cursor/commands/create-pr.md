# Create a pull request

Help me open a pull request from the current branch.

1. **Check state**: Confirm we’re on a feature branch (not `main`/`master`) and that all commits are pushed.
2. **Summarize**: Briefly describe what this branch changes (from commit history and/or diff).
3. **Draft PR content**: Provide:
   - **Title**: Short, imperative summary (e.g. "Add Cursor commands for commit, PR, and push").
   - **Description**: What changed, why, and any notes for reviewers. Mention if docs or tests were updated.
4. **Open the PR**: Give the exact `gh pr create` (or equivalent) command with title and body, or step-by-step instructions for creating the PR in the GitHub UI.

Follow the repo’s PR guidelines: include appropriate commit messages in the branch, update documentation for new features, and call out any breaking changes.
