# Create a commit

Help me create a single, well-scoped commit from the current working tree.

1. **Review changes**: List what’s modified (e.g. `git status` / diff) and group into one logical change.
2. **Propose a commit message** that:
   - Uses imperative mood (e.g. "Add cursor commands" not "Added cursor commands").
   - Is clear and concise (under ~72 characters for the subject).
   - Optionally includes a short body if the change needs explanation.
3. **Stage and commit**: Run the exact `git add` and `git commit` commands so I can execute them (or run them for me if you have permission).

If there are unrelated changes, suggest how to split into separate commits or ask which changes should be included in this commit.
